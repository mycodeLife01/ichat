## 数据库结构摘要

- `users`：保存用户名、邮箱、密码哈希、邮箱验证状态、启用状态和时间戳。
- `refresh_tokens`：保存 refresh token 哈希、过期时间和撤销时间。
- `email_verification_tokens`：预留邮箱验证 token 哈希、过期时间和使用时间。
- `conversations`：保存用户拥有的 conversation、标题、软删除时间和时间戳。
- `messages`：保存线性 conversation message，支持 `user` 和 `assistant` 角色、conversation 内 `position`、软归档时间，以及可选来源 `run_id`。
- `runs`：保存 LLM run 状态机、provider 信息、错误信息、usage metadata、worker lease/heartbeat 和生命周期时间戳。
- `run_events`：保存 run 内按 `seq` 递增的可重放事件流，payload 使用 PostgreSQL `JSONB`。

## 关键约束和索引

- `users` 对 `lower(username)` 和 `lower(email)` 建唯一索引。
- `messages` 对 `(conversation_id, position)` 建唯一约束，保证同一 conversation 内线性顺序唯一。
- `runs` 对 active 状态建立 partial unique index：同一 conversation 同时只能有一个 `queued`、`started`、`streaming` 或 `cancelling` run。
- `runs` 建 `(status, created_at)` 索引，用于后续 worker claim queued run。
- `runs` 建 `lease_expires_at` 索引，用于后续 lease timeout recovery。
- `run_events` 对 `(run_id, seq)` 建唯一约束和 replay 索引。
- `messages.role`、`runs.status`、`run_events.type` 使用字符串字段加 `CHECK` 约束，不使用 PostgreSQL enum。
